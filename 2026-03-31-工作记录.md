### 任务 2: 修复 Vue 语法渲染问题

**问题现象：** 项目页面显示源码而非渲染内容，用户看到 `v-if`、`:href` 等 Vue 语法

**小爪诊断：**
- `projects/index.md` 使用 `<script setup>` 语法，VitePress 1.x 未正确编译
- Vue 语法被当作纯文本输出

**尝试方案 A：** 添加 `markdown.vueComponent: true` 配置
- **结果：** 无效

**尝试方案 B：** 重构为独立 Vue 组件
1. 创建 `.vitepress/theme/components/ProjectsGrid.vue`
2. 移动 `_data/downloads.json` 到 `.vitepress/_data/`
3. 修改 `projects/index.md` 使用 `<ProjectsGrid />` 组件
- **结果：** 问题依旧，组件内仍用 `<script setup>`

**最终方案：** 改用 Options API
- 将 `ProjectsGrid.vue` 中的 `<script setup>` 改为 Options API 格式
- 保留语言切换功能
- **结果：** ✅ 成功

### 任务 4: 强制锁定暗色主题

**问题现象：**
- 主题切换按钮仍然存在
- 用户可切换到亮色主题，左侧边栏变白、右侧内容黑

**小爪诊断：** 切换按钮未隐藏，浏览器可手动切换主题

**最终方案（组合拳）：**

1. **配置层：** `appearance: false`（完全禁用主题切换）
   - 添加到全局 `themeConfig`
   - 添加到 `locales.root.themeConfig`
   - 添加到 `locales.zh.themeConfig`

2. **CSS 层：** 强制覆盖所有主题变量
   ```css
   :root, :root.light, :root.dark, html:not(.dark), html.dark {
     --vp-c-bg: #0d1117 !important;
     /* ... 所有变量都用 !important */
   }
   ```

3. **DOM 层：** 隐藏切换按钮
   ```css
   .VPNavBarAppearance,
   .VPSwitchAppearance,
   .appearance-action {
     display: none !important;
   }
   ```

**结果：** ✅ 完全成功

**提交记录：**
- `921ade4` - Fix: Completely disable theme switching, force dark mode
- `3b9c6ab` - Fix: Hide theme switcher in config and CSS

## 🎯 经验总结

### VitePress 1.x 兼容性
1. `<script setup>` 在 `.vue` 单文件组件中支持有限
2. 使用 Options API 更稳定可靠

### 主题控制
1. `appearance: 'dark'` 只是默认值，不隐藏切换按钮
2. `appearance: false` 才能完全禁用
3. 需要在所有 locale 中分别配置
4. CSS 强制覆盖 (`!important`) 是必要的兜底方案
